Implementation notes

This page is intentionally simple and stable so the auth/product foundation is clean before more features are layered on top.

Correct URL target

Use workspace.html in the repo and nav links. A bare /workspace route will 404 unless you add routing support.

Redirect behavior

If a user is not signed in, this page sends them to signin.html?next=workspace.html, so the post-auth destination is explicit and stable.

Data dependency

The page reads from users, organization_members, organizations, and diagnostic_runs. If row-level policies block reads, the workspace will still load but show an explanatory message.
