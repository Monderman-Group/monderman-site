Each version looks at the same structural issue from a different vantage point. Pick the lane that best matches what you can see firsthand.

You can change lanes later by starting over. The underlying diagnostic engine remains the same; the questions shift to fit what each perspective can observe most credibly.

Operational

Built for people closest to the work. Best for surfacing duplicate entry, manual workarounds, process density, and where the operating environment quietly consumes effort.

Managerial

Built for people coordinating teams and execution. Best for identifying reporting burden, systems friction, recurring review load, and where administrative work is crowding out operating work.

Executive

Built for leaders shaping operating design and governance. Best for assessing process accumulation, control creep, redesign readiness, and where administrative overhead is weakening system responsiveness.

Step 2Choose how deep you want this run to go.

Choose diagnostic depth

Pick the shortest run that still fits the importance of the workflow. For most first passes, the 30-minute version is the best starting point.

Shorter runs still deliver useful signal. Longer runs simply uncover more hidden patterning and stronger opportunity framing over time.

Why time depth matters

  • Get a baseline: understand how the pathway works today
  • Re-run later: measure whether changes actually improved the workflow
  • Compare perspectives: see where different roles experience the pathway differently
  • Track drift: longer runs surface accumulating burden more clearly
Best starting pointUse the 30-minute diagnostic when the goal is a strong first read without adding unnecessary friction to the run.
